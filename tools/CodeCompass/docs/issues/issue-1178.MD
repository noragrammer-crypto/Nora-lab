# #1178 [Story] CodeCompass: ホットスポット判定とランキング出力

## 本文

## 概要

変更頻度（#1176）と複雑度スコア（#1177）を統合し、
「(変更頻度 × 複雑度) / LOC」の密度スコアでホットスポットをランキングし、
人間可読・機械可読の両形式で出力する。CodeCompass のレイヤー2（ホットスポット判定）の中核。

## 背景

CodeScene の核心ロジック。`/ LOC` を入れることで大規模ファイルの過大評価を防ぎ、
「触るほど複雑なコード」を最優先のリファクタリング候補として特定する。

## やること

- #1176（変更頻度）と #1177（複雑度）の出力データを統合する
- `(変更頻度 × 複雑度) / LOC` で密度スコア（リファクタリングROI）を算出する
- 密度スコア降順でランキングした `hotspots.md`（人間が読む用）を出力する
- 同データを `hotspots.json`（xp_Architect に渡す機械可読データ）として出力する
- 出力先は `codecompass/` 配下とし、リポジトリにコミット可能な形式にする

## 受け入れ条件

- [ ] 変更頻度データと複雑度データを統合して密度スコアを算出できる
- [ ] 密度スコア降順でホットスポットランキングを生成できる
- [ ] `codecompass/hotspots.md`（人間用）を出力できる
- [ ] `codecompass/hotspots.json`（機械可読・xp_Architect連携用）を出力できる
