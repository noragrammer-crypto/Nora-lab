# #1179 [Story] CodeCompass: 依存グラフ生成（CodeGraph統合）

## 本文

## 概要

ホットスポット判定（#1178）で抽出した上位ファイルに対して、
CodeGraph（npm, MIT）を用いた caller/callee 依存グラフを生成し、Mermaid形式で出力する。
CodeCompass のレイヤー1後段（依存関係の可視化）に当たる。

## 背景

全ファイルの依存グラフ生成はコスト爆発の原因になるため、
親イシュー #727 の方針どおり「ホットスポット上位（目安20%）のみ CodeGraph で深掘りする」。
AST解析・依存グラフ生成自体は CodeGraph の `GraphTraverser` をそのまま流用し、自前実装は最小化する。

## やること

- CodeGraph（npm パッケージ）を導入し、ホットスポット上位ファイルの caller/callee を解析する
- 依存関係を Mermaid 形式（`dependency-graph.mmd`）で出力する
- 出力先は `codecompass/` 配下とし、リポジトリにコミット可能な形式にする

## 受け入れ条件

- [ ] CodeGraph を使って JS/Python コードの caller/callee 依存関係を解析できる
- [ ] ホットスポット上位ファイルのみを対象に依存グラフを生成できる（全件展開はしない）
- [ ] 依存グラフを Mermaid 形式（`codecompass/dependency-graph.mmd`）で出力できる

## スコープ外

- 全ファイルに対する依存グラフ生成（コスト爆発のため非対応）
- xp_Architect へのリファクタリング提案生成（→ 別ストーリー）
