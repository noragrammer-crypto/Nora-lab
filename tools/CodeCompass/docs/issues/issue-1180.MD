# #1180 [Story] CodeCompass: xp_Architect連携によるリファクタリング提案生成

## 本文

## 概要

蒸留済みデータ（ホットスポットスコア + 依存グラフ + git履歴）を構造化JSONとして
xp_Architect に渡し、リファクタリング設計提案（`actions.md`）を自動生成する。
CodeCompass のレイヤー3（AIアーキテクト連携・SKILL.md化）に当たる、エピックの最終出口。

## 背景

「コードそのものは渡さず、構造化JSONのみ渡す」ことで判断ブレを防止する方針（親イシュー #727）。
これにより「分析→提案→行動」のループが閉じ、SoloXP（xp_Director → xp_Architect → xp_Implementer → xp_Auditor）
への統合が可能になる。

## やること

- `hotspots.json`（#1178）と `dependency-graph.mmd`（#1179）を統合した構造化JSONを構築する
  ```json
  {
    "file": "path/to/file.js",
    "hotspot_score": 0.82,
    "complexity": 34,
    "changes_90d": 120,
    "callers": 8,
    "callees": 12
  }
  ```
- 構造化JSONを xp_Architect に渡し、リファクタリング設計提案（パターン提示）を取得する
- 提案結果を `codecompass/actions.md`（リファクタリングアクション一覧）として出力する
- SKILL.md として呼び出し可能な形に整備する

## 受け入れ条件

- [ ] ホットスポットスコア + 依存グラフ情報を統合した構造化JSONを生成できる
- [ ] 構造化JSON（コードそのものは含まない）を xp_Architect に渡してリファクタリング提案を得られる
- [ ] 提案結果を `codecompass/actions.md` として出力できる
- [ ] SKILL.md として独立呼び出し可能な形に整備されている

## スコープ外

- xp_Implementer によるリファクタリング実施（提案までがスコープ。実施は別途イシュー化して通常のSoloXPフローに乗せる）
- レビューワークフロー統合（feature-dev連携、セキュリティレビュー追加等は親イシューコメントの「将来検討」扱いとし対象外）
