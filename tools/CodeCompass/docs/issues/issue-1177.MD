# #1177 [Story] CodeCompass: 複雑度スコアリングエンジン（AST解析）

## 本文

## 概要

AST（抽象構文木）解析により、JS/Python ファイルの複雑度スコアを算出する。
CodeCompass の蒸留エンジン（レイヤー1）の2本目の柱で、ホットスポット判定の入力データとなる。

## 背景

CodeScene の核心ロジックである「変更頻度 × 複雑度」のうち、複雑度側の自前実装。
親イシュー #727 で「flog/reek相当を制御フローカウントで実装する」と整理されている部分。
パーサー自体は自作せず、既存ASTライブラリを流用する方針（JS: ESLint/Babel AST、Python: 標準 `ast` モジュール）。

## やること

- JS ファイルを ESLint/Babel AST で解析し、制御フロー（if/for/while/switch等）の出現数をカウントする
- Python ファイルを標準 `ast` モジュールで解析し、同様に制御フローをカウントする
- ファイルごとの複雑度スコアと LOC（行数）を構造化データ（JSON）として出力する

## 受け入れ条件

- [ ] JS ファイルの制御フロー（if/for/while等）をAST解析でカウントし複雑度スコアを算出できる
- [ ] Python ファイルも同様に複雑度スコアを算出できる
- [ ] ファイルごとの LOC（行数）も合わせて取得できる
- [ ] 複雑度スコア + LOC を構造化データ（JSON）として出力できる

## スコープ外

- 変更頻度の集計（→ 別ストーリー、並行実施可）
- caller/callee 依存グラフ生成（→ 別ストーリー）
- TypeScript / Ruby 等 Phase 2 以降の言語対応

### 中程度・低リスク指摘

#### [Medium] 変更頻度集計でリネームファイルがファントムキーになる

**ファイル**: `CodeCompass/lib/change-frequency.js:5` (NUMSTAT_LINE regex)

`git log --numstat` はファイルリネーム時に `old/file.js => new/file.js` 形式で出力する。
現状の正規表現はこの文字列全体（`old/file.js => new/file.js`）をファイルキーとして取り込むため、
- 実際のファイルパスとは一致しないキーがMapに登録される
- リネーム後の新ファイルの変更回数が過小カウントされる

影響: ホットスポット精度の低下（データ品質バグ）。クラッシュはしない。

```js
// 対応案: リネーム行を検出してスキップ or 新パスのみ取り出す
const match = NUMSTAT_LINE.exec(line);
if (!match) continue;
const file = match[3].includes('=>') ? match[3].split('=>').pop().trim() : match[3];
```

---

#### [Medium] `python3` が未インストールの環境では ENOENT で解析全体がクラッシュする

**ファイル**: `CodeCompass/lib/complexity-score.js:97` (countPythonComplexities)

`python3` が存在しない（最小 CI コンテナ等）場合、`execFileSync` が ENOENT をスローし
`analyzeComplexity` 全体が中断する。Python ファイルを含むリポジトリではツールが全く動作しない。

影響: 環境依存で実行不可。ユーザー向けエラーメッセージなし。

```js
// 対応案: ENOENT をキャッチして Python ファイルのみスキップ
try {
  return new Map(Object.entries(JSON.parse(raw)));
} catch (e) {
  console.warn('[warn] python3 unavailable — Python files skipped');
  return new Map(absolutePaths.map((p) => [p, 0]));
}
```

---

#### [Low] `--days=NaN` がエラーなしに空リストを返す

**ファイル**: `CodeCompass/scripts/change-frequency.js:parseArgs`

`--days=abc` のような非数値を渡すと `Number('abc') === NaN` となり、
`git --since=NaN.days` が git に渡される。git はこれを無効な日付として扱い、
コミットを一切返さないため、空の JSON 配列が出力される。エラーや警告はなし。

影響: 無言で空結果になる。CLIで低確率のユーザーミス。
